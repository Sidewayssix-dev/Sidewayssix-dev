## 🤝 Contribution Guidelines
We welcome contributions that align with our vision. Here's how to get involved:

<details>
<summary><strong>Step-by-Step Guide to Contributing</strong></summary>

1. **Fork the Repo:** Click the fork button on any of our repos (e.g., [sideways-six-mvp](https://github.com/sideways-six/sideways-six-mvp)).
2. **Clone Locally:** `git clone https://github.com/YOUR-USERNAME/sideways-six-mvp.git`
3. **Create a Feature Branch:** `git checkout -b feat/your-amazing-idea`
4. **Make Changes:** Code away! Ensure tests pass and docs update.
5. **Commit Conventionally:** `git commit -m "feat: add autonomous agent endpoint"`
6. **Push & PR:** `git push origin feat/your-amazing-idea` then open a Pull Request.
7. **Review Process:** Expect feedback; we require 2+ approvals for merges.
8. **Post-Merge:** We'll handle the rest—changelogs auto-generate!

- **Code Style:** ESLint for JS, Black for Python.
- **Testing:** 80% coverage minimum via Jest/Pytest.
- **Issues:** Label with `good-first-issue` for beginners.

See our full [CONTRIBUTING.md](CONTRIBUTING.md) and [Code of Conduct](CODE_OF_CONDUCT.md).
